✅ تعليمات ربط صفحات الموقع بلوحة التحكم
🛠️ المرحلة 1: إعداد لوحة التحكم الرئيسية
المسار: /admin

نظام الدخول: حماية بسيطة في البداية باستخدام كلمة مرور (admin123) عبر localStorage.

لاحقًا يتم استبدالها بـ JWT.

الهيكل الأساسي:

شريط علوي يحتوي على اسم الموقع وزر تسجيل خروج.

قائمة جانبية تحتوي على التبويبات التالية:

📌 تبويبات لوحة التحكم ومهام كل واحد
1. 🛠️ تبويب: إعدادات الموقع (site)
الهدف: التحكم بالمحتوى العام الظاهر في الصفحة الرئيسية والتذييل.

 تعديل اسم التطبيق (appName) الظاهر في الهيدر.

 رفع الشعار واستبداله.

 تعديل نبذة الصفحة الرئيسية أو النصوص التعريفية.

 تعديل محتوى التذييل:

"من نحن"، "سياسة الخصوصية"، "اتصل بنا"، "تغيير اللغة"، "حقوق النشر".

2. 👥 تبويب: إدارة المستخدمين (users)
الهدف: التحكم الكامل في معلومات المستخدمين.

 عرض جميع المستخدمين (الاسم، البريد، النجوم، المستوى).

 تعديل بيانات الحساب (اسم، صورة، نقاط، كروت).

 تعديل عدد الكروت المجانية والمدفوعة.

 تعديل المستوى والنجوم يدويًا عند الحاجة.

 عرض آخر الألعاب الخاصة بالمستخدم.

3. 📚 تبويب: إدارة الفئات (categories)
الهدف: التحكم في فئات الأسئلة التي تظهر أثناء إنشاء اللعبة.

 عرض الفئات الحالية (الاسم، الأيقونة، عدد الأسئلة).

 إضافة فئة جديدة (اسم + أيقونة).

 تعديل / حذف الفئات.

 التحكم في حالة الفئة:

"ظاهرة فقط إذا كانت تحتوي على عدد كافٍ من الأسئلة (مثلاً 12)".

4. ✏️ تبويب: إدارة الأسئلة (questions)
الهدف: إدارة قاعدة بيانات الأسئلة كاملة.

 عرض قائمة الأسئلة مع إمكانية الفلترة (حسب الفئة، الصعوبة...).

 إضافة سؤال جديد (نص / صورة / فيديو / PDF).

 تعديل وحذف الأسئلة.

 عرض عدد المرات التي تم فيها استخدام السؤال.

5. ⚙️ تبويب: إعدادات اللعبة (gameSettings)
الهدف: التحكم الكامل في كيفية سير اللعبة.

 تحديد عدد الفرق (الحد الأدنى والأقصى).

 تحديد عدد الفئات المسموح بها في كل لعبة.

 ضبط زمن الإجابة الأولى والثانية (بالثواني).

 تفعيل أو تعطيل وسائل المساعدة:

مثل: "تخطي"، "عكس الدور"، "خصم نقاط".

 التحكم في عرض سجل اللعبة، وإعدادات ظهور النتائج.

6. 🃏 تبويب: إدارة باقات الكروت (packages)
الهدف: التحكم بالباقات التي يمكن للمستخدم شراؤها.

 عرض قائمة الباقات (الاسم، عدد الكروت، السعر).

 تعديل أو حذف باقة.

 تحديد نوع الكروت (مجانية / مدفوعة).

 تحديد مدة صلاحية الكروت المجانية (أسبوع، شهر...).

 ضبط سياسة تصفير وتجديد الكروت المجانية تلقائيًا.

7. ⭐ تبويب: إدارة المستويات (levels)
(يمكن إضافته لاحقًا)
الهدف: ربط عدد النجوم بالمستوى، ومنح مكافآت حسب المستوى.

 أسماء المستويات (برونزي، ذهبي، بلاتيني...).

 عدد النجوم المطلوب للوصول لكل مستوى.

 مكافآت المستوى (نقاط / كروت شهرية).

 إعداد شروط انخفاض المستوى تلقائيًا في حالة الخمول.

🔚 ملاحظات ختامية:
يجب ربط كل تبويب بـ API حقيقية تحفظ البيانات في قاعدة البيانات.

يتم تحميل البيانات عند الدخول للتبويب، مع استخدام Toast للتنبيهات.

يتم استخدام Dialog أو Drawer للتعديل السريع داخل اللوحة.

جميع الحقول النصية تقبل اللغة العربية بشكل كامل.
